## ─── 1. Executive Summary ───────────────────────────────────────────────

Modern satellite tracking, command, and telemetry networks demand 99.999% availability. However, unexpected physical infrastructure cuts, BGP routing configuration leaks, and hardware degradation often go undetected until a system failure interrupts ground-to-space links. 

As a cybersecurity founder and B.Tech student, I've spent the last couple of years analyzing network anomalies and perimeter defense systems. When I read Challenge 13, I realized that typical solutions using heavy Python frameworks or cloud-dependent LLMs simply won't cut it in secure, air-gapped ground stations. If an AI NOC copilot relies on external internet calls or causes memory leaks that crash the monitoring agent, it becomes a liability rather than an asset.

**VIGIL (Verified Intelligent Ground-station Infrastructure Liaison)** is my solution: a predictive, AI-assisted NOC copilot built entirely in memory-safe Rust. VIGIL runs 100% offline within air-gapped mission control centers. It ingests, validates, analyzes, and explains complex network telemetry anomalies in real-time, matching high-performance statistical detection (Z-Score + IQR + RoC) and unsupervised Machine Learning (Isolation Forest) with local, sandboxed LLM diagnostics and a deterministic expert rule fallback.

## ─── 2. Key Technical Features & Capabilities ────────────────────────────

### 1. Cryptographic Ingestion Gate
- All telemetry streams (OSPF neighbor changes, SNMP trap alerts, MPLS paths, interface metrics) are validated using SHA-256 HMAC tokens.
- Invalid or unauthenticated sources are blocked before parsing to protect against perimeter breaches.

### 2. Multi-Metric Hybrid Anomaly Detection
- Analyzes latency, bandwidth, prefix advertisements, and packet loss statistics.
- Processes features over circular memory-bounded windows using:
  - **Z-Score**: Identifies deviation from dynamic statistical baselines.
  - **Interquartile Range (IQR)**: Detects sudden spike outliers.
  - **Rate of Change (RoC)**: Highlights sharp, short-term trends.

### 3. Air-Gapped AI Diagnostics Copilot
- Integrates local llama.cpp or Ollama endpoints to run small, instruction-tuned LLMs offline (such as Mistral-7B or Phi-3.5).
- Features a **Rule-Based Expert Fallback** to provide uninterrupted service if model files are missing or systems are running with limited CPU resources.

### 4. Zero-Trust Mitigation Safety Sanitizer
- Intercepts raw text generated by the local LLM.
- Redacts shell redirections, script tags, and system binary executions (e.g., `sudo`, `rm`) using compiled regular expressions, preventing prompt injection attacks from reaching operators.

### 5. High-Availability Web Dashboard
- Features custom Canvas-based real-time telemetry sparklines.
- Includes a live telemetry ingestion viewer, an anomaly console, and a scenario injection controller.
- Dynamically loads failure playbooks and scenarios (e.g., Fiber Cut, Route Leak, Degraded Transceiver, Congestion Buildup) at the push of a button.

### Detailed Fault Scenarios & Predictive Lead Time

1. **Progressive Congestion Buildup**: Generates a slow ramp-up of interface utilization on ground station routers. As utilization climbs past 70% and 80%, the system begins to detect rising latency and minor packet drops. This allows VIGIL to raise "Low" and "Medium" warning thresholds long before a complete service disruption.
2. **BGP Route Flap**: Simulates BGP peering session state transitions (Established -> Active -> Down). The anomaly detection engine flags prefix drift and neighbor flaps, while the AI Copilot outlines route redundancy loss.
3. **Intermittent MPLS Underlay Failure**: Models fluctuating underlay link degradation where packet loss and latency spike intermittently, simulating physical transmission issues or optical degradation.
4. **Tunnel/SD-WAN Degradation**: Gradually introduces CRC optical interface errors and packet loss, showing predictive lead-time metrics on the dashboard to allow preventative maintenance.

## ─── 4. Solution Differentiators ────────────────────────────────────────

1. **Zero-Unsafe, Hardened Rust Stack**: The entire codebase enforces `#![deny(unsafe_code)]`. As someone running a security startup (xcomrade.tech), I know that 70% of high-severity CVEs come from memory mismanagement. By using Rust, VIGIL guarantees complete freedom from buffer overflows, data races, and use-after-free conditions.
2. **True Air-Gapped Autonomy**: The system requires zero external API keys, zero cloud access, and zero internet telemetry. Telemetry database runs locally on `redb`, features are processed in-memory using sliding circular buffers, and AI analysis is run on-device using llama.cpp/Ollama GGUF execution.
3. **Hybrid Ensemble Outlier Detection**: Instead of relying solely on statistics (which have high false-positive rates) or heavy deep-learning ML models (which struggle to run on low-power NOC terminals), VIGIL implements an ensemble score. It fuses Z-Score, IQR, and Rate of Change calculations with an unsupervised **Isolation Forest** ML model (via `linfa`).
4. **Resiliency Expert Playbook Fallbacks**: If the local model is missing or the GPU/CPU is throttled, VIGIL automatically switches to deterministic, rule-based expert playbooks. A NOC operator will never stare at a blank dashboard or get a "server error" during a critical fiber-cut incident.
5. **No Bloat Web Operations**: The dashboard avoids heavy frameworks like React or Tailwind, using pure HTML5, CSS3, and a vanilla JavaScript frontend. It renders low-overhead telemetry sparklines dynamically onto HTML5 canvas surfaces, compiling the entire web server and asset bundles right into the compiled daemon.
6. **Zero-Hassle Automated Deployment**: A highly polished `install.sh` wizard auto-detects system capabilities (Rust, Ollama, existing models). It offers one-click downloads of lightweight models (like Qwen2.5-1.5B) with live progress bars, configures paths automatically, and provides instant fallback mechanisms, making it accessible for operators of any technical level to launch the suite in under 5 minutes.

## ─── 5. Future Roadmap ──────────────────────────────────────────────────

1. **SNMP Trap Receiver**: Add a native UDP listener to ingest SNMP v2c/v3 traps directly from network equipment, removing the need for an external SNMP manager.
2. **Federated Multi-Site Deployment**: Extend the architecture to support multiple VIGIL instances across geographically distributed ground stations, with an aggregation layer that correlates anomalies across sites.
3. **Model Fine-Tuning Pipeline**: Build tooling to fine-tune the local LLM on historical incident reports specific to the deployment site, improving diagnosis accuracy over time.
4. **Hardware Security Module (HSM) Integration**: Replace the TPM attestation stub with a production HSM interface for cryptographic key storage and model integrity verification.
5. **Alert Escalation & Notification**: Integrate with local email relays or serial-connected alert panels for automated escalation of critical anomalies to on-call personnel.
